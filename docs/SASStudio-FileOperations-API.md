# SAS Studio File Operations API Documentation

> **Note**: This documentation was generated by exploring SAS Studio 3.8 via browser automation. The API is internal/undocumented and may vary between versions.

## Overview

SAS Studio provides a RESTful API for file and folder operations through the `/sasexec` endpoint. This API enables programmatic access to the SAS Content Server and file system.

## Base Configuration

```python
BASE_URL = "http://192.168.0.141/SASStudio/38"
API_URL = f"{BASE_URL}/sasexec"
SESSION_ID = "cf689d9e-cc5a-4039-93e8-326548b5ad5f"  # Remote session ID

# Required headers for all requests
headers = {
    "RemoteSession-Id": SESSION_ID,
}

# Cookies:
# - Dev instance (192.168.0.141): No authorization cookie required.
#   The server sets JSESSIONID in the session-creation response; capture it
#   and include it only for the /reset endpoint.
# - Production instances: An authorization token cookie from the SAS Studio
#   login flow is required for ALL requests, including session creation.
cookies = {
    "JSESSIONID": "75947C89AE5F3110867C7D04204566D4",  # Only needed for /reset on dev; required everywhere on production
}
```

## Authentication & CSRF Protection

### CSRF Token Acquisition

SAS Studio uses CSRF tokens for state-changing operations. Acquire tokens on startup:

```python
# Get CSRF tokens
response = session.head(
    url=f"{BASE_URL}/sasexec/csrf",
    cookies=cookies,
)
csrf_header = response.headers.get("X-CSRF-HEADER")
csrf_token = response.headers.get("X-CSRF-TOKEN")

# Include in subsequent write operations
headers_with_csrf = {
    **headers,
    "X-CSRF-HEADER": csrf_header,
    "X-CSRF-TOKEN": csrf_token,
}
```

## Path Encoding

The API uses special encoding for paths to avoid URL parsing issues:

| Encoding | Meaning                 | Example                                           |
| -------- | ----------------------- | ------------------------------------------------- |
| `~ps~`   | Path separator (`/`)    | `/folders/myfolders` → `~ps~folders~ps~myfolders` |
| `~dot~`  | Period (`.`)            | `file.sas` → `file~dot~sas`                       |
| `~sg~`   | Library/table separator | Used for SAS datasets (e.g., `SASHELP~AIR~`)      |
| `~` (trailing) | Path terminator    | Used for **file metadata** only, not directories  |

### Important: Trailing Tilde Usage

- **Directory listing**: NO trailing `~` — e.g., `~ps~folders~ps~myfolders`
- **File metadata**: WITH trailing `~` — e.g., `~ps~folders~ps~myfolders~ps~test~dot~sas~`
- **Library/table**: Uses `~` as separator — e.g., `SASHELP~AIR~`

### Path Encoding Algorithm

```python
import urllib.parse

def encode_path(path, is_ftp=False):
    """
    Encode a file path for use in SAS Studio API URLs.

    Standard paths use double URI encoding with ~ps~ for slashes.
    FTP paths have special handling.
    """
    if is_ftp:
        # FTP paths have special encoding rules
        return get_encoded_ftp_path(path)

    # Replace / with ~ps~
    encoded = path.replace("/", "~ps~")

    # Replace . with ~dot~
    encoded = encoded.replace(".", "~dot~")

    # Apply double URI encoding
    encoded = urllib.parse.quote(urllib.parse.quote(encoded, safe="~"), safe="~")

    # Additional special character handling
    encoded = encoded.replace("&", "%26")
    encoded = encoded.replace("#", "%23")
    encoded = encoded.replace(",", "%252C")
    encoded = encoded.replace(";", "%253B")
    encoded = encoded.replace("+", "%252B")
    encoded = encoded.replace("?", "%3F")

    return encoded

def decode_path(encoded_path):
    """Decode a SAS Studio encoded path back to normal form."""
    decoded = encoded_path.replace("~ps~", "/")
    decoded = decoded.replace("~dot~", ".")
    return urllib.parse.unquote(decoded)
```

## File Tree Loading Process

The SAS Studio file tree uses a hierarchical loading pattern that starts from the root node (`_root_`) and loads children on-demand.

### Tree Loading Sequence

```
1. GET /{sessionId}/_root_
   → Returns root with top-level items (Folder Shortcuts, My Folders, etc.)

2. GET /{sessionId}/noChildren/_root_
   → Returns root metadata without children array

3. When user expands "My Folders":
   GET /{sessionId}/~ps~folders~ps~myfolders?time={timestamp}
   → Returns My Folders contents with children array

4. For deeper levels, repeat step 3 with the appropriate encoded path
```

### Step 1: Load Root Node

**Endpoint**: `GET /sasexec/{sessionId}/_root_`

```python
response = session.get(
    url=f"{API_URL}/{SESSION_ID}/_root_",
    cookies=cookies,
    headers=headers,
)
root_data = response.json()
```

**Response Structure**:

```json
{
  "id": "_root_",
  "name": "localhost",
  "isDirectory": true,
  "uri": "/",
  "uriParent": "",
  "children": [
    {
      "id": "Folder Shortcuts",
      "name": "Folder Shortcuts",
      "isDirectory": true,
      "uri": "/Folder Shortcuts",
      "haveChildren": false
    },
    {
      "id": "My Folders",
      "name": "My Folders",
      "isDirectory": true,
      "uri": "/folders/myfolders",
      "haveChildren": true
    }
  ]
}
```

### Step 2: Load Root Without Children (Optional)

Get root metadata without the children array:

**Endpoint**: `GET /sasexec/{sessionId}/noChildren/_root_`

### Step 3: Load Folder Contents on Expand

**Endpoint**: `GET /sasexec/{sessionId}/{encodedPath}?time={timestamp}`

```python
from datetime import datetime

# Load contents of /folders/myfolders with cache-busting timestamp
timestamp = datetime.utcnow().strftime("%a, %d %b %Y %H:%M:%S GMT")
encoded_path = "~ps~folders~ps~myfolders"

response = session.get(
    url=f"{API_URL}/{SESSION_ID}/{encoded_path}",
    params={"time": timestamp},
    cookies=cookies,
    headers=headers,
)
folder_data = response.json()
```

**Response Data Model**:

```json

  "stub": false,
  "asciiEbcdicConversionRequired": false,
  "id": "~ps~folders~ps~myfolders",
  "name": "myfolders",
  "isDirectory": true,
  "uri": "/folders/myfolders",
  "uriParent": "/folders/",
  "children": [
    {
      "stub": false,
      "id": "~ps~folders~ps~myfolders~ps~hello_world.sas",
      "name": "hello_world.sas",
      "isDirectory": false,
      "uri": "/folders/myfolders/hello_world.sas",
      "uriParent": "/folders/myfolders",

      "size": "166 bytes",
      "editable": true,
      "isPartitionedDatasetMember": false,
      "isPDS": false,
      "isGitRepository": false

}
```

## File Content Operations

### Read File Content

Retrieves the text content of a file.

**Endpoint**: `GET /sasexec/sessions/{sessionId}/workspace/{filePath}`

```python
file_path = "/folders/myfolders/hello_world.sas"
response = session.get(
    url=f"{API_URL}/sessions/{SESSION_ID}/workspace/{file_path}",
    cookies=cookies,
    headers=headers,
)
content = response.text  # Returns raw file content
```

### Write/Update File Content

Saves content to a file (creates or overwrites).

**Endpoint**: `POST /sasexec/sessions/{sessionId}/workspace/{filePath}`

```python
file_path = "/folders/myfolders/hello_world.sas"
content = """
/* Simple SAS Program */
data hello;
  message="Hello from SAS Studio!";
  date_created=today();
  format date_created date9.;
  output;
run;
"""

response = session.post(
    url=f"{API_URL}/sessions/{SESSION_ID}/workspace/{file_path}",
    cookies=cookies,
    headers={
        **headers_with_csrf,  # Include CSRF tokens
        "Content-Type": "text/file",
    },
    data=content,
)
```

**Notes**:

- Content-Type should be `text/file` for file uploads
- Creates file if it doesn't exist, overwrites if it does
- CSRF tokens required for POST operations

### Create Folder

Creates a new folder within a parent directory.

**Endpoint**: `PUT /sasexec/{sessionId}/{parentPath}`

```python
# Create folder "new_folder" in /folders/myfolders
response = session.put(
    url=f"{API_URL}/{SESSION_ID}/~ps~folders~ps~myfolders",
    cookies=cookies,
    headers={
        **headers_with_csrf,
        "Content-Type": "application/json",
    },
    json={
        "stub": False,
        "id": "~ps~folders~ps~myfolders",
        "name": "myfolders",
        "isDirectory": True,
        "uri": "/folders/myfolders",
        "children": [
            # Include existing children plus new folder
            {
                "stub": False,
                "id": "~ps~folders~ps~myfolders~ps~new_folder",
                "name": "new_folder",
                "isDirectory": True,
                "uri": "/folders/myfolders/new_folder",
                "uriParent": "/folders/myfolders",
            }
        ]
    }
)
```

**Note**: The PUT request must include all existing children plus the new folder. SAS Studio uses a "replace entire directory listing" approach.

## Important Notes

1. **Session Management**: The `RemoteSession-Id` header must match an active SAS Studio session
2. **Authentication**: On **production** instances, an authorization token cookie from the SAS Studio login flow is required for **all** requests. On the **dev instance**, no authorization cookie is needed — the `JSESSIONID` cookie (set during session creation) is only required for the `/reset` endpoint.
3. **Path Encoding**: Use `~ps~` instead of `/` when referring to paths in tree endpoints
4. **Double Encoding**: Most paths are double URI-encoded using `encodeURI(encodeURI(path))`
5. **Folder Creation**: PUT operations replace the entire directory listing, so include all existing items
6. **CSRF Protection**: Write operations (POST, PUT, DELETE) require CSRF tokens
7. **Version Compatibility**: This API was observed on SAS Studio 3.8 - other versions may differ
8. **z/OS Special Handling**: MVS datasets and PDS members have special encoding and validation rules

## Error Handling

Common HTTP status codes:

- `200 OK` - Success
- `201 Created` - Resource created successfully
- `204 No Content` - Operation completed successfully (no response body)
- `400 Bad Request` - Invalid request parameters
- `404 Not Found` - File/folder doesn't exist
- `409 Conflict` - Resource already exists or conflict in operation
- `500 Internal Server Error` - Server error or invalid request

Always check `response.raise_for_status()` when using Python requests.
